---
description: Maintain and update the project documentation in the `documentation/` folder.
---

1.  **Identify the Change**: Determine if the recent code changes affect the system architecture, API, setup process, or database schema.
2.  **Locate Documentation**: Check the `documentation/` folder for relevant files:
    - `project_overview.md`: High-level goals and scope.
    - `architecture.md`: System design, components, and data flow.
    - `api_specs.md`: API endpoints and usage (if manual).
    - `setup_guide.md`: detailed setup instructions.
    - `database_schema.md`: Schema design and decisions.
3.  **Update or Create**:
    - If a relevant file exists, update it to reflect the new state.
    - If the change represents a new domain or significant feature not covered, create a new markdown file in `documentation/`.
4.  **Formatting**: Ensure all documentation uses clear Markdown formatting with headers, code blocks, and diagrams (Mermaid) where appropriate.
5.  **Review**: Verify that the documentation is consistent with the `README.md` and other root-level files.
